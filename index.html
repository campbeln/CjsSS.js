Overview

CjsSS (Cascading Javascript Style Sheet) is LESS SASSy and more Stylush than the competition ;)

  • Browser Support: IE8+ (or IE6+ with optional cjsss.polyfill.js or jQuery)
  • Licensing Fees: $0, nada, nutin'.. but Nick does accept tips; licensed under the MIT License
  • Installation: n/a, just include 
    cjsss.min.js
    (Node.js optional for server-side preprocessing)
  • Tiny! Only ~3,300 bytes compressed
  • Near-zero learning curve; it's just Javascript embedded within CSS!
  • Your CSS stays CSS, no language features to learn or mentally decode
  • Vanilla Javascript toolkit with no external dependencies
  • No need for developer debugging tools, just view source or inspect element
  • Create complex mixins with nested Javascript 
    object
    s
  • Client-side CSS preprocessor and reprocessor capabilities with no plugins required
  • camelCase
    to 
    dash-case
    of Javascript 
    object
    keys converted auto-magicially
  • eval
    uated Javascript fully sandboxed by default
  • Use any Javascript library within your CSS, not just language supported functions
  • ~550 lines of source code is fully commented and accessible to the brave and not-so-brave alike

Options

Option values can be passed into CjsSS.js at a number of points: 
  1. <script> cjsss.process("[cjsss]", { crlf: "\n" }); </script>

    overrides: 
  2. <style type="text/css" cjsss='{ "crlf": "\r\n" }' />
      or   
    <link rel="stylesheet" type="text/css" href="css/site.css" cjsss='{ "crlf": "\r\n" }' />

    override:
  3. The default values described below (which can be overridden).
In the example above,

scope

STRING (Enumeration, see below) sandbox The scope that the CjsSS.js-embedded Javascript will be 
eval
uated within: 
  • sandbox
    - isolated scope. Code is evaluated within a detached 
    IFRAME
    with zero access to any of the host page's resources.* 
  • local
    - non-"use strict" closure scope. Code is evaluated within the host page with full access to global resources, but once execution is complete embedded Javascript resources are no longer accessable (unless explicitly set within the global scope). 
  • global
    - non-"use strict" global scope. Code is evaluated within the host page's global scope with full access to its global resources, allowing embedded Javascript resources to persist within the global scope.

    object

  • cjsss.process(vElements, oOptions)

    vElements

    oOptions
    object
    in preference to the default values set within 
    cjsss.options

  • cjsss.services

How To Override The Default Option Values

If you wish to override any of the default option values, you must include a Javascript code block like the one below before CjsSS.js's 
SCRIPT
tag: 
window.cjsss = {
    options: {
        key: value
    }
};
Where 
key
equals the option value you wish to reset and 
value
equals the new default value.

Getting Started

Assuming the default option values are in effect as described above:
  1. Download CjsSS.js From GitHub
  2. Include the CjsSS.js 
    SCRIPT
    s in the 
    HEAD
    of your HTML document below any 
    LINK
    / 
    STYLE
    tags you want to process: 
    <script type="text/javascript" src="js/cjsss-polyfill.js"></script> <!-- IE6-7 support -->
<script type="text/javascript" src="js/cjsss.min.js"></script>
  3. Add the 
    cjsss
    attribute to the 
    LINK
    and/or 
    STYLE
    tags you want to process: 
    <link rel="stylesheet" type="text/css" href="css/site.css" cjsss>
<style type="text/css" cjsss>...</style>
  4. Define the 
    options
     used by CjsSS.js (if any) using JSON-style object definitions (both keys and values are "double quoted"): 
    <style type="text/css" cjsss='{ "crlf": "\n" }'>...</style>
<link rel="stylesheet" type="text/css" href="css/site.css" cjsss='{ "crlf": "\n" }'>
  5. Edit the CSS within the 
    LINK
    and/or 
    STYLE
    tags to use CjsSS.js like this: 
    /*<script>
    var mainColor = 'blue';
</script>*/

    color: /*mainColor*/;
}
NOTE: You are also permitted to use the attribute name 
data-cjsss
in place of 
cjsss

Gotchas

  • If you are using the default 
    autorun
    behavior of CjsSS.js then you will want to include the CjsSS.js 
    SCRIPT
    reference after all 
    cjsss
    attributed 
    LINK
    / 
    STYLE
    tags.
  • As 
    LINK
    tag CSS files are read via AJAX, the limitations of 
    XMLHttpRequest
     can restrict where you serve your CSS files from. If your CSS files exist on the same server, this is a non-issue. 
  • LINK
    tags are removed from the DOM, their CSS file contents are modified and placed within a new 
    STYLE
    tag that is appended into the 
    HEAD
    . This process can break some relative URLs within your CSS files (e.g. 
    background-image: url("../img/a.png");
    ). It is therefore recommended that you use absolute URLs within your CSS files (e.g. 
    background-image: url("/img/a.png");
    ) in order to avoid any such errors.
  • Embedded 
    SCRIPT
    s are all evaluated in series at initilization prior to CSS modification, so the following will not work as expected: 
    /*<script>
    var mainColor = 'blue';
</script>*/
DIV {
    color: /*mainColor*/; /* expecting 'blue' but gets 'green' */
}

/*<script>
    mainColor = 'green';
</script>*/
SPAN {
    color: /*mainColor*/; /* expecting 'green' */
}
    At initilization 
    mainColor
    will be set to "blue" then "green" prior to any CSS modification, so the 
    color
    of both DIV and SPAN will be set to "green".
  • IE8+ is supported by CjsSS.js out-of-the-box, with support back to IE6 once 
    document.querySelectorAll
    and 
    JSON.parse
    functionality has been polyfilled via 
    cjsss.polyfill.js
    or with jQuery (which is auto-magicially imported) included prior to CjsSS.js. The required "advanced" JavaScript functionality is:
    • document.querySelectorAll*, 
      cjsss.polyfill.js
      or jQuery's 
      $()
      (* - basic support in IE8 is sufficient)
    • JSON.parse*, 
      cjsss.polyfill.js
      or jQuery's 
      $.parseJSON()
      (* - support from IE8)
    • XMLHttpRequest or IE6-7's ActiveXObject
    • getElementsByTagName, createElement, getAttribute, and setAttribute - basic support in IE6-7 is sufficient

Side Effects

  • LINK
    tags are removed from the DOM, their CSS file contents are modified and placed within a new 
    STYLE
    tag that is appended into the 
    HEAD
    . This is done so that the original unprocessed CSS referenced by the 
    LINK
    tag does not apply to the webpage.
  • If you have elected to 
    expose
    CjsSS.js    , the 
    window.cjsss
    object will be added under the global scope.
  • If you have elected to have a 
    scope
    of either 
    global
    or 
    local
    , CjsSS.js-embedded Javascript has the ability to modify the global scope.
  • If you have elected to have a 
    scope
    of 
    global
    , the 
    window.mixin
    function will be added under the global scope (though it will not overwright an existing 
    window.mixin
    if one exists).
  • And of course... the CSS within CjsSS.js attributed 
    LINK
    and 
    STYLE
    tags is modified based on the embedded Javascript.

What is up with the snake?

Thanks to the akwardness of the acronym (ending in three S's) it seemed reminisent of a snake's hiss. So... "CSS and Javascript sss-ing"... "CJ hissing"... Hey! I need a two-headed snake!
